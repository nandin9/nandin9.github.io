Shader：星云shader分析

2025-06-08 | Shader,图形图像,渲染

使用光线步进（Raymarching）和分形迭代技术生成复杂的3D分形结构。以下是对代码的详细解释：

  1. 宏定义和全局设置
    glsl

#define rot(a) mat2(cos(a),sin(a),-sin(a),cos(a)) // 2D旋转矩阵宏
定义一个用于2D旋转的矩阵宏，简化旋转变换操作。

  1. 主函数初始化
    glsl
    void mainImage(out vec4 O, vec2 u) {
    float t = iTime*.01 + .25; // 时间变量（缩放+偏移）
    float s = .1; // 光线步进初始步长
    float f = 1.; // 颜色衰减因子
    float a, l; // 分形迭代累加器/临时距离

    vec3 R = iResolution; // 分辨率
    vec3 p; // 当前点坐标
    vec3 D = vec3((u - .5R.xy)/R.x.4, .5); // 光线方向
    vec3 o = vec3(1,.5,.5) + vec3(t+t, t, -2); // 摄像机位置（随时间移动）

    O -= O; // 清零输出颜色

光线方向计算：

将像素坐标 u 从屏幕空间转换到标准化设备坐标（中心为原点）。

D.z = 0.5 控制视角倾斜角度。

摄像机动画：

摄像机位置 o 在X、Y轴随时间平移，Z轴固定为 -2（向后）。

  1. 场景旋转
    glsl
    mat2 r1 = rot(.5); // 创建旋转矩阵（0.5弧度）
    D.xz *= r1; // 旋转光线方向
    o.xz *= r1; // 旋转摄像机位置
    对场景的XZ平面进行旋转，增加动态效果。

  2. 主循环：光线步进 + 分形迭代
    glsl
    for (int i, r = 0; r++ < 20; ) {
    // 1. 光线步进并折叠空间
    p = abs(mod(o + s * D, 1.7) - .85);

    // 2. Kleinian分形迭代 (17次)
    a = t = 0.;
    for (i = 0; i++ < 17; ) {
    l = length(p); // 计算当前距离
    a += abs(l - t); // 累加距离变化量
    t = l; // 保存旧距离
    p = abs(p) / dot(p,p) - .53; // 核心分形公式
    }

    // 3. 计算分形特征强度
    a *= a * a; // 立方放大细节

    // 4. 颜色衰减控制
    if (r > 7) f = min(1., .7 + aa*.001);

    // 5. 累加颜色 (RGB通道独立计算)
    O.rgb += f + s * vec3(1, s, sss) * a * .0015 * f;

    // 6. 更新步进参数
    f *= .73; // 颜色衰减
    s += .1; // 增加步长

}
核心分形公式解析：
glsl
p = abs(p) / dot(p,p) - .53
abs(p)：折叠空间到第一象限，创造对称性。

/ dot(p,p)：球面反演（球心在原点），将远离中心的点拉近。

  • .53：平移创造自相似结构。

迭代17次形成无限递归的细节。

颜色累加技巧：
vec3(1, s, sss)：

R通道：基础白色

G通道：随步长 s 变绿

B通道：s³ 产生蓝色渐变

a*.0015*f：用分形特征强度调制颜色

s * …：距离越远颜色贡献越大

  1. 后处理与输出
    glsl
    O = .0085 * O + .0015 * length(O);
    0.0085 * O：基础亮度缩放

0.0015 * length(O)：根据整体亮度添加泛光效果

使亮区产生光晕，增强立体感

Metal Learning：创建自定义的Metal View

1. 准备渲染目标

在 Metal 下，最典型的做法是在自定义的 NSView（或 UIView）里把它的 layerClass 指定为 CAMetalLayer。这样一旦每帧需要绘制时，就能拿到一个 CAMetalDrawable，它自带一个 texture，就是本次渲染的“离屏纹理”，待渲染结束后由系统自动提交到屏幕。

  • 要指定 CAMetalLayer.pixelFormat
  • 通过 [metalLayer nextDrawable] 拿到一个 CAMetalDrawable，它的 texture 会成为本次渲染目标

AppKit/UIKit/SwiftUI...

2024-10-03 | macos,ios,swiftui

AppKit 、UIKit和SwiftUI ， Apple 提供的三种不同的框架，用于构建客户端UI，面向不同的开发场景和目标设备。

  1. AppKit：macOS 的原生 UI 框架
    特点：
  • 面向对象：以面向对象的方式组织 UI 组件（如 NSWindow、NSButton）。
  • 功能强大：提供了 macOS 特有的窗口管理和高级用户界面功能（如菜单栏、工具栏、键盘快捷键支持）。
  • 历史悠久：自 macOS 的前身 NeXTSTEP 系统开始就存在，是成熟且功能丰富的框架。
  • 低级控制：开发者需要手动管理窗口和视图的生命周期。
  • 适用场景：构建 macOS 原生应用程序。
